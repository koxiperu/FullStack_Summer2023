JS Team Project
tiny.cc/js-project1
The goal of the exercise is to reproduce the mechanics of "duck hunt" game with a player who controls the duck and the other who has to shoot it.
Video
You will have to work on two computers in a team of two sharing code with GIT (and CI on Netlify if possible)
Keep thinking as a team with patience and kindness. A whiteboard is at your disposal for some kind explanations.
In case of hard to reach goal, do not hesitate to use the panic button to contact the senior developer.
You will create a netlify website (sync with the git repository) so that other team can playwith it.
1 - Travel Management
The cube must be able to move throughout the window using the arrow keys.
Visual Algorythm
Help
2 - Beware of collisions
The cube shall only move in a playground. create a container that will constrains the movement of the cube. When he reaches the bottom, it teleports up.
Visual Algorythm
Help
3 - Add the player to the mouse
When the mouse clicks on the cube:
The cube becomes red
event listening stops (keyboard and mouse).
alert: The game is over: it's victory!
Help
4 - Victory case for the keyboard player
We will allow the keyboard player to win if he manages to escape the player mouse over 30 sec. So add a timer to games that win the player on the keyboard.
After 30 seconds, the keyboard user wins. setTimeout() / setInterval() cancel the timeout using clearTimeout()
The cube becomes green
Event listeners are disabled
Alert: the game is over, victory of the player at the keyboard
5 - Score
Both players have a score and have 5 match.
Ask each player his name
Create two HTML elements to display the name and score
Create a variable to store the score of each player
Increase this value with each victory and update the HTML area
If the score is 3, a congratulation message appears.
